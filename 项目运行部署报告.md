# ydbi-demucs 项目运行部署报告

## 1. 项目职责

`ydbi-demucs` 是 YouBi 流水线中的音频分离服务，对应阶段名为 `demucs`，数据库阶段表为 `demucs`。服务轮询数据库中 `status = 'ready'` 的任务，读取 downloader 阶段产出的原始音频，调用 Demucs 模型分离人声与背景音，并把结果写回 MySQL 与 MinIO。

在全链路阶段中的位置：

```text
downloader -> demucs -> whisper -> translator -> speaker -> combiner -> uploader
```

成功后会将下一阶段 `whisper.status` 从 `pending` 推进到 `ready`。

## 2. 当前运行环境

- Python 版本：`>=3.12`
- 包管理：标准 Python packaging，项目定义在 `pyproject.toml`
- 启动入口：`ydbi-demucs = ydbi_demucs.main:main`
- 工作目录：默认 `/tmp/ydbi/demucs/<task_id>`，可通过 `YDBI_DEMUCS_WORK_DIR` 覆盖
- 轮询间隔：`10` 秒
- 运行设备：`DEVICE = "auto"`，优先 `cuda`，其次 `mps`，最后 `cpu`
- Demucs 代码来源：默认使用当前服务仓库的 `submodule/demucs`，可通过 `YDBI_DEMUCS_REPO` 指向其他 Demucs checkout
- 默认 Demucs 模型：`htdemucs_ft`
- 默认 shifts：`1`

服务需要能够访问：

- MySQL：当前配置指向远端 `youbi` 数据库
- MinIO：当前配置指向远端 `ydbi` bucket
- Demucs 子模块目录
- 本机或容器内可写的临时工作目录

## 4. 环境依赖

`pyproject.toml` 声明的 Python 运行依赖：

```text
dora-search>=0.1.12

mysql-connector-python>=9.0.0
numpy

```

代码运行时依赖 Demucs 子模块提供的 `demucs.api.Separator` 和 `save_audio`。因此实际运行环境还需要满足 Demucs 及其底层音频依赖，通常包括：

- `torch`
- `torchaudio`
- `torchcodec`，当前 `torchaudio.save()` 会通过 TorchCodec 写出 WAV，缺失时会在分离完成后保存音频失败
- Demucs 子模块依赖
- 可用的音频解码能力，常见为 `ffmpeg`

如果当前仓库的 `submodule/demucs` 不存在，服务会报错：

```text
Demucs is not available. Run: git submodule update --init --recursive
```

如需复用其他仓库中的 Demucs checkout，可设置：

```bash
export YDBI_DEMUCS_REPO="/path/to/demucs"
```

## 5. 数据流与产物

### 输入

服务从 `demucs` 查询 ready 任务，并合并 `video_info` 中的视频信息。音频输入优先级：

1. 如果 downloader 阶段的 `operator` 与当前服务相同，并且 `audio_source_path` 存在，则直接使用本地文件
2. 否则使用 `audio_source_url` 从 MinIO 下载
3. 如果没有 URL 但本地路径存在，则使用本地路径
4. 如果都不可用，任务失败

### 输出

Demucs 产物位于任务工作目录：

```text
<work_dir>/<task_id>/media/audio_vocals.wav
<work_dir>/<task_id>/media/audio_bgm.wav
```

上传到 MinIO：

```text
<task_id>/demucs/audio_vocals.wav
<task_id>/demucs/audio_bgm.wav
```

写回字段：

- `audio_vocals_path`
- `audio_vocals_url`
- `audio_bgm_path`
- `audio_bgm_url`

这些字段会写入 `video_info`，非共享字段会写入当前阶段表。

## 6. 数据库表与状态推进

服务使用的核心表：

- `task`
- `demucs`
- `whisper`
- `downloader`
- `video_info`
- `service_heartbeat`

状态流转：

1. `demucs.status = ready`
2. 服务抢占任务，更新为 `running`
3. 成功后更新为 `success`
4. 推进 `whisper.status = ready`
5. 失败后更新 `demucs.status = failed`，同时更新 `task.status = failed`

服务会自动补充 `operator` 字段，并维护心跳表 `service_heartbeat`。

## 7. 本地部署手册

### 1. 准备 Python 环境

```bash
cd /Users/hoshuuch/Money/YouBi/services/demucs
git submodule update --init --recursive
```

```text
submodule/demucs
```

`pyproject.toml` 已声明 Demucs API 所需的 Python 依赖。系统层还需要安装 `ffmpeg`，用于音频解码。

```bash

```

或：

```bash
python -m ydbi_demucs.main
```

- `video_info.audio_bgm_url`

确认以下对象存在：

```text
s3://ydbi/<task_id>/demucs/audio_vocals.wav
s3://ydbi/<task_id>/demucs/audio_bgm.wav
```

## 9. 常见问题

- Demucs 子模块不存在：初始化 `/Users/hoshuuch/Money/YouDub-webui/submodule/demucs`
- 输入音频不存在：检查 `video_info.audio_source_path` 和 `audio_source_url`
- 跨机器执行找不到本地文件：确保上游已上传 MinIO，或让同一 operator 机器处理
- GPU 不可用：`auto` 会降级到 `cpu`，但速度会明显变慢
- MinIO 下载失败：检查 URL 是否能解析为 `ydbi` bucket 内对象

## 10. 当前风险与改进建议

- 数据库密码和 MinIO 密钥硬编码在源码中，建议改为环境变量或密钥管理
- Demucs 子模块路径硬编码到本机目录，跨机器部署需要调整
- `pyproject.toml` 未显式声明 Demucs、torch、torchaudio 等实际推理依赖，建议补齐或提供部署脚本
- 当前没有 Dockerfile，建议补充容器化部署定义，减少机器环境差异
